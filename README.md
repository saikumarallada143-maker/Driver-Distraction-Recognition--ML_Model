# Driver-Distraction-Recognition--ML_Model
Deep learning repository for identifying distracted drivers, including safe driving, texting, and fatigue

A computer-vision project that classifies in-car driver images into **10 behavior
categories** using a **ResNet-50** convolutional neural network built from scratch
with Keras / TensorFlow.  
Model selection uses **Leave-One-Group-Out (LOGO) cross-validation** grouped by driver,
so the model is always evaluated on a driver it has never seen during training — a
much more realistic test of generalization than a random train/val split.

---

## Classes

| Label | Behavior                        |
|-------|---------------------------------|
| c0    | Safe driving                    |
| c1    | Texting — right hand            |
| c2    | Talking on the phone — right    |
| c3    | Texting — left hand             |
| c4    | Talking on the phone — left     |
| c5    | Operating the radio             |
| c6    | Drinking                        |
| c7    | Reaching behind                 |
| c8    | Hair and makeup                 |
| c9    | Talking to passenger            |

## Pipeline

```
Load CSV metadata
       │
       ▼
Quick EDA (class balance, images-per-subject, null check)
       │
       ▼
Shuffle & encode class labels (c0-c9 → 0-9)
       │
       ▼
Convert images to 64×64×3 numpy arrays (cached as .npy)
       │
       ▼
Build ResNet-50 from scratch (identity + convolutional blocks)
       │
       ▼
LOGO cross-validation (grouped by driver/subject)
   2 epochs → 5 epochs → 10 epochs  (bias/variance analysis)
       │
       ▼
Train final model on full training set (10 epochs)
       │
       ▼
Predict on holdout/test set → save probabilities CSV
```

## Dataset

This project uses the **State Farm Distracted Driver Detection** dataset from Kaggle:

🔗 https://www.kaggle.com/c/state-farm-distracted-driver-detection

After downloading, organize it as follows:

```
data/
├── driver_imgs_list.csv       # provided by Kaggle
├── test_file_names.csv        # provided by Kaggle
└── imgs/
    ├── train/                 # training images
    └── test/                  # holdout / test images
```

Update `DATA_DIR` at the top of the notebook if you prefer a different layout.

## Setup

### 1. Clone the repository

```bash
git clone <your-repo-url>
cd <your-repo-name>
```

### 2. Create a virtual environment

```bash
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

### 3. Download the dataset

Download from Kaggle (link above) and place files under `data/` as shown above.

### 4. Launch the notebook

```bash
jupyter notebook Distracted_Driver_Detection.ipynb
```

Run cells top-to-bottom. The first time you run **Section 5**, image arrays are built
and cached — subsequent runs skip this step automatically.

## Results

| Epochs | Train Acc | Val Acc | Bias  | Variance |
|--------|-----------|---------|-------|----------|
| 2      | ~31 %     | ~25 %   | High  | Low      |
| 5      | ~38 %     | ~26 %   | High  | Growing  |
| 10     | ~87 %     | ~41 %   | Low   | High     |

**Key finding:** The model overfits significantly at 10 epochs (86 % train vs 41 % val
accuracy). The validation score stabilizes while training accuracy keeps climbing — a
classic high-variance / overfitting pattern calling for regularization and/or
data augmentation before further scaling.

## Improvements / Next Steps

- **Data augmentation** — random horizontal flips, rotations, brightness/contrast jitter
  via `tf.keras.preprocessing.image.ImageDataGenerator` or Keras augmentation layers.
- **Transfer learning** — fine-tune a pretrained ResNet-50 / MobileNetV2 / EfficientNet
  (ImageNet weights); likely to outperform training from scratch on this dataset size.
- **Regularization** — add dropout layers, weight decay, and/or label smoothing.
- **Callbacks** — `EarlyStopping`, `ReduceLROnPlateau`, and `ModelCheckpoint` for more
  stable training.
- **Experiment tracking** — log runs with TensorBoard or Weights & Biases.
- **Richer evaluation** — per-class precision/recall/F1 using `sklearn.metrics.classification_report`.
- **Deployment** — wrap inference in a FastAPI endpoint or a Streamlit demo app.
